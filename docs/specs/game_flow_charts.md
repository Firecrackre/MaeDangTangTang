## 1. 개요

**게임 흐름**은 플레이어가 게임을 시작한 뒤 아이템을 구매하고, 큐브를 사용해 잠재능력을 확인한 뒤, 판매하거나 한 번 더 사용하기를 선택하며 빚 상환을 버텨내는 전체 진행 구조를 의미한다.

본 문서는 각 기획서에서 정의한 세부 규칙이 실제 플레이 안에서 어떤 순서로 연결되는지 정리한다.

세부 확률, 판매가 수식, 스크래치 조작 방식 등은 개별 문서에서 다루며, 
이 문서에서는 **플레이 흐름 / 상태 전환 / 주요 판단 지점 / 외부 연동**을 중심으로 정리한다.

---

## 2. 목표

게임 흐름의 목표는 다음과 같다.

| 목표 | 설명 |
| --- | --- |
| 핵심 루프 정리 | 아이템 구매부터 판매, 빚 상환까지의 반복 구조를 명확히 한다. |
| 판단 지점 명확화 | 플레이어가 언제 판매할지, 언제 큐브를 한 번 더 사용할지 고민하게 되는 지점을 정리한다. |
| 시스템 연결 정리 | 큐브 스크래치, 잠재능력 판정, 판매가 계산, 재화, 빚 상환의 연결 순서를 정리한다. |
| 게임 오버 조건 명확화 | 파괴, 메소 부족, 빚 상환 실패 등 실패 흐름을 정리한다. |
| 구현 기준 제공 | 각 상태에서 어떤 UI와 버튼이 활성화되는지 기준을 제공한다. |

---

## 3. 전체 핵심 흐름

게임의 기본 흐름은 다음과 같다.

```
1. 게임 시작
2. 시작 메소 지급
3. 기본 아이템 선택 및 구매
4. 큐브 사용
5. 잠재능력 3줄 내부 생성
6. 큐브 스크래치 팝업 표시
7. 잠재능력 3줄 순차 공개
8. 옵션 / 이탈 / 파괴 여부 확인
9. 판매가 계산
10. 판매하기 또는 한 번 더 사용하기 선택
11. 큐브 사용 횟수 누적
12. 일정 큐브 사용 횟수마다 빚 상환 발생
13. 상환 성공 시 다음 플레이 계속
14. 상환 실패 시 게임 오버
```

---

## 4. 플레이 상태 구분

게임 진행은 다음 상태로 구분한다.

| 상태 | 설명 |
| --- | --- |
| 대기 상태 | 플레이어가 다음 행동을 선택할 수 있는 기본 상태 |
| 아이템 선택 상태 | 큐브를 사용할 기본 아이템을 선택하는 상태 |
| 큐브 사용 상태 | 선택한 아이템에 큐브를 사용하는 상태 |
| 스크래치 상태 | 잠재능력 3줄을 긁어 확인하는 상태 |
| 결과 확인 상태 | 3줄 결과와 예상 판매가를 확인하는 상태 |
| 선택 상태 | 판매하기 또는 한 번 더 사용하기를 선택하는 상태 |
| 판매 완료 상태 | 아이템을 판매하고 메소를 획득한 상태 |
| 파괴 상태 | 아이템이 파괴되어 판매할 수 없는 상태 |
| 빚 상환 상태 | 정해진 큐브 사용 횟수에 도달해 빚을 갚는 상태 |
| 게임 오버 상태 | 빚 상환 실패 등으로 게임이 종료된 상태 |

---

## 5. 게임 시작 흐름

게임을 시작하면 플레이어는 기본 메소를 가진 상태로 시작한다.

```
1. 게임 시작
2. 시작 메소 지급
3. 현재 큐브 사용 횟수 초기화
4. 다음 빚 상환까지 남은 큐브 개수 표시
5. 기본 아이템 선택 가능 상태로 전환
```

### 시작 시 초기값

| 항목 | 내용 |
| --- | --- |
| 보유 메소 | 40000 메소 |
| 큐브 사용 횟수 | 0회 |
| 현재 아이템 | 없음 |
| 다음 상환까지 남은 큐브 개수 | 초기 상환 주기에 따라 표시 |

---

## 6. 아이템 선택 흐름

플레이어는 보유 메소를 사용해 큐브를 사용할 기본 아이템을 선택한다.

현재 기준 장비 종류는 다음 5종이다.

| 장비 종류 | 유효 스탯 | 유효 공격 계열 |
| --- | --- | --- |
| 검 | STR | 공격력, 보스 공격력 |
| 활 | DEX | 공격력, 보스 공격력 |
| 스태프 | INT | 마력, 보스 공격력 |
| 완드 | INT | 마력, 보스 공격력 |
| 단도 | LUK | 공격력, 보스 공격력 |

### 아이템 선택 흐름

```
1. 구매 가능한 기본 아이템 목록 표시
2. 플레이어가 아이템 선택
3. 보유 메소가 구매 비용 이상인지 확인
4. 구매 가능하면 메소 차감
5. 현재 아이템으로 등록
6. 큐브 사용 가능 상태로 전환
```

### 실패 처리

| 상황 | 처리 |
| --- | --- |
| 보유 메소 부족 | 아이템 구매 불가 |
| 현재 아이템 존재 | 새 아이템 구매 불가 또는 교체 확인 필요 |
| 아이템 데이터 없음 | 구매 버튼 비활성화 |

---

## 7. 큐브 사용 흐름

플레이어는 현재 아이템에 큐브를 사용해 잠재능력을 재설정한다.

```
1. 필요 메소 보유 여부 확인
2. 큐브 사용 가능 여부 확인
3. 큐브 사용 실행
4. 큐브 비용 차감
5. 큐브 사용 횟수 1 증가
6. 잠재능력 판정 실행
7. 잠재능력 3줄 결과 생성
8. 큐브 스크래치 팝업 표시
```

### 큐브 사용 시 처리되는 값

| 항목 | 처리 |
| --- | --- |
| 큐브 비용 | 재화 로직에서 차감 |
| 큐브 사용 횟수 | 1 증가 |
| 잠재능력 결과 | 새로 생성 |
| 기존 잠재능력 | 한 번 더 사용하기 시 사라짐 |
| 빚 상환 카운트 | 큐브 사용 횟수 증가에 따라 갱신 |

### 큐브 사용 기준

- 큐브 사용은 현재 아이템이 있을 때만 가능하다.
- 큐브를 사용할 때마다 큐브 사용 횟수가 증가한다.
- 한 번 더 사용하기를 선택하면 기존 잠재능력과 판매가는 사라지고 새 결과가 생성된다.

---

## 8. 잠재능력 판정 흐름

큐브를 사용하면 내부적으로 잠재능력 판정이 진행된다.

```
1. 큐브 종류 확인
2. 잠재능력 등급 결정
3. 옵션 1 생성
4. 옵션 2 생성
5. 옵션 3 생성
6. 각 옵션 타입 판정
7. 주요 옵션일 경우 이탈 여부 판정
8. 파괴 항목 포함 여부 확인
9. 조합 판정
10. 결과 데이터를 큐브 스크래치 화면으로 전달
```

### 전달 데이터

| 데이터 | 사용처 |
| --- | --- |
| 등급 | 판매가 계산, 연출 |
| 옵션 1~3 | 큐브 스크래치 표시 |
| 옵션 타입 | 공개 피드백, 판매가 계산 |
| 이탈 여부 | 공개 피드백, 판매가 계산 |
| 파괴 여부 | 파괴 처리, 버튼 상태 |
| 조합 판정 | 결과 연출, 판매가 계산 |

---

## 9. 큐브 스크래치 흐름

잠재능력 결과는 즉시 공개되지 않고, 은박으로 가려진 상태로 표시된다.

```
1. 큐브 스크래치 팝업 열림
2. 잠재능력 3줄이 은박으로 가려진 상태로 표시
3. 플레이어가 1번째 줄을 긁음
4. 1번째 옵션 공개 및 피드백 출력
5. 현재 예상 판매가 임시 갱신
6. 플레이어가 2번째 줄을 긁음
7. 2번째 옵션 공개 및 피드백 출력
8. 현재 예상 판매가 임시 갱신
9. 플레이어가 3번째 줄을 긁음
10. 3번째 옵션 공개 및 피드백 출력
11. 3줄 결과 최종 판정
12. 최종 판매가 또는 Destroyed 표시
13. 버튼 상태 갱신
```

### 스크래치 중 버튼 상태

| 상태 | 판매하기 | 한 번 더 사용하기 | 확인 |
| --- | --- | --- | --- |
| 스크래치 전 | 비활성화 | 비활성화 | 비활성화 |
| 1줄 공개 | 비활성화 | 비활성화 | 비활성화 |
| 2줄 공개 | 비활성화 | 비활성화 | 비활성화 |
| 3줄 공개 완료 / 정상 | 활성화 | 활성화 | 선택적으로 활성화 |
| 3줄 공개 완료 / 파괴 | 비활성화 | 비활성화 | 활성화 |

---

## 10. 판매가 계산 흐름

판매가 계산은 공개된 잠재능력 결과와 현재 장비 종류를 기준으로 진행한다.

```
1. 현재 아이템의 장비 종류 확인
2. 현재 아이템의 기본 판매가 확인
3. 잠재능력 3줄 확인
4. 각 옵션이 현재 장비에 유효한지 판정
5. 옵션 보너스 총합 계산
6. 유효 옵션 조합 보너스 계산
7. 등급 배수 적용
8. 유효 이탈 옵션 개수에 따른 이탈 배수 적용
9. 파괴 여부 확인
10. 최종 판매가 산정
11. 큐브 스크래치 화면에 판매가 전달
```

### 판매가 계산 기준

```
최종 판매가 =
(아이템 기본 판매가 + 옵션 보너스 총합 + 조합 보너스) × 등급 배수 × 이탈 배수
```

단, 파괴 항목이 포함되어 있으면 판매가 계산을 진행하지 않고 **Destroyed**로 표시한다.

### 유효 옵션 기준

| 장비 종류 | 유효 스탯 | 유효 공격 계열 |
| --- | --- | --- |
| 검 | STR | 공격력, 보스 공격력 |
| 활 | DEX | 공격력, 보스 공격력 |
| 스태프 | INT | 마력, 보스 공격력 |
| 완드 | INT | 마력, 보스 공격력 |
| 단도 | LUK | 공격력, 보스 공격력 |

### 계산 흐름 기준

- 잠재능력 판정에서 대박 조합으로 분류되었더라도, 판매가 계산에서는 현재 장비와 맞는 유효 옵션만 조합 보너스와 이탈 배수에 반영한다.
- 비유효 주요 옵션은 낮은 보너스로 처리한다.
- 잡옵은 매우 낮은 보너스로 처리한다.
- 파괴는 모든 계산보다 우선한다.

---

## 11. 결과 선택 흐름

3줄이 모두 공개되고 파괴가 발생하지 않았다면, 플레이어는 판매하기 또는 한 번 더 사용하기를 선택할 수 있다.

```
1. 최종 판매가 표시
2. 판매하기 버튼 활성화
3. 한 번 더 사용하기 버튼 활성화
4. 플레이어 선택 대기
```

### 선택지

| 선택 | 처리 |
| --- | --- |
| 판매하기 | 현재 판매가만큼 메소 획득 |
| 한 번 더 사용하기 | 같은 아이템에 큐브를 다시 사용 |
| 확인 | 팝업 종료 또는 결과 확인 |

---

## 12. 판매 흐름

플레이어가 판매하기를 선택하면 현재 아이템을 판매하고 메소를 획득한다.

```
1. 판매하기 버튼 클릭
2. 확정 판매가 확인
3. 현재 아이템 제거
4. 판매가만큼 메소 지급
5. 현재 판매가 초기화
6. 큐브 스크래치 팝업 종료
7. 다음 아이템 선택 상태로 전환
```

### 판매 처리 기준

- 판매는 파괴되지 않은 아이템에만 가능하다.
- 판매 후 현재 아이템은 사라진다.
- 판매 수익은 보유 메소에 즉시 반영된다.
- 판매 후 플레이어는 새 아이템을 구매해 다음 사이클을 진행한다.

---

## 13. 한 번 더 사용하기 흐름

플레이어가 한 번 더 사용하기를 선택하면 같은 아이템에 큐브를 다시 사용한다.

```
1. 한 번 더 사용하기 버튼 클릭
2. 큐브 사용 가능 여부 확인
3. 큐브 비용 차감
4. 큐브 사용 횟수 1 증가
5. 기존 잠재능력 결과 제거
6. 기존 예상 판매가 초기화
7. 새로운 잠재능력 판정 실행
8. 큐브 스크래치 흐름 재진입
```

### 한 번 더 사용하기 처리 기준

- 한 번 더 사용하기는 아이템이 파괴되지 않았을 때만 가능하다.
- 한 번 더 사용하기를 선택하면 기존 잠재능력 결과는 사라진다.
- 한 번 더 사용하기를 선택하면 기존 예상 판매가는 보장되지 않는다.
- 큐브 사용 횟수는 한 번 더 사용하기 시에도 증가한다.
- 큐브 사용 횟수 증가에 따라 빚 상환이 발생할 수 있다.

---

## 14. 파괴 흐름

잠재능력 3줄 중 파괴 항목이 포함되어 있으면 아이템은 파괴된다.

```
1. 파괴 항목 공개
2. 파괴 피드백 출력
3. 예상 판매가를 Destroyed로 표시
4. 판매하기 버튼 비활성화
5. 한 번 더 사용하기 버튼 비활성화
6. 확인 버튼 활성화
7. 플레이어가 확인 버튼 클릭
8. 파괴된 아이템 제거
9. 큐브 스크래치 팝업 종료
10. 다음 아이템 선택 상태로 전환
```

### 파괴 처리 기준

- 파괴는 모든 좋은 옵션과 등급보다 우선한다.
- 파괴된 아이템은 판매할 수 없다.
- 파괴된 아이템에는 큐브를 다시 사용할 수 없다.
- 파괴 시 아이템 구매 비용과 큐브 사용 비용은 회수되지 않는다.
- 파괴 결과에서도 큐브 사용 횟수는 이미 증가한 상태로 유지된다.

---

## 15. 빚 상환 흐름

빚 상환은 큐브 사용 횟수를 기준으로 발생한다.

```
1. 큐브 사용 횟수 증가
2. 다음 상환까지 남은 큐브 개수 갱신
3. 상환 조건 도달 여부 확인
4. 상환 조건에 도달하면 빚 상환 팝업 표시
5. 보유 메소가 상환 금액 이상인지 확인
6. 상환 가능하면 메소 차감
7. 다음 상환 조건 갱신
8. 상환 불가하면 게임 오버
```

### 상환 처리 기준

| 상황 | 처리 |
| --- | --- |
| 보유 메소 충분 | 상환 금액 차감 후 게임 계속 |
| 보유 메소 부족 | 게임 오버 |
| 아이템 보유 중 상환 발생 | 상환 먼저 처리 후 진행 |
| 스크래치 중 상환 조건 도달 | 큐브 사용 처리 이후 적절한 시점에 상환 팝업 표시 |

### 상환 발생 시점

빚 상환은 큐브 사용 횟수 증가 후 조건을 확인한다.

다만 큐브 스크래치 도중 흐름이 끊기지 않도록, 실제 상환 팝업은 다음 중 하나의 시점에 표시한다.

| 표시 시점 | 설명 |
| --- | --- |
| 큐브 사용 직후 | 큐브 사용과 동시에 상환 조건을 확인하고 즉시 표시 |
| 스크래치 결과 완료 후 | 스크래치 3줄 공개가 끝난 뒤 상환 팝업 표시 |

메커톤 버전에서는 플레이 흐름이 끊기지 않도록 **스크래치 결과 완료 후 상환 팝업 표시**를 우선 고려한다.

## 16. 게임 오버 흐름

게임 오버는 빚 상환 실패 시 발생한다.

```
1. 빚 상환 시점 도달
2. 상환 금액 확인
3. 보유 메소 확인
4. 보유 메소가 부족하면 상환 실패
5. 게임 오버 연출 출력
6. 최종 결과 표시
7. 재시작 또는 종료 선택
```

### 게임 오버 조건

| 조건 | 처리 |
| --- | --- |
| 빚 상환 금액 부족 | 게임 오버 |
| 더 이상 아이템을 구매할 수 없음 | 후순위로 게임 오버 조건 검토 |
| 큐브 사용 비용 부족 | 큐브 사용 불가, 즉시 게임 오버는 아님 |

### 게임 오버 표시 정보

| 정보 | 설명 |
| --- | --- |
| 최종 보유 메소 | 게임 종료 시점의 보유 메소 |
| 총 큐브 사용 횟수 | 플레이어가 사용한 큐브 횟수 |
| 총 판매 횟수 | 판매에 성공한 아이템 수 |
| 최고 판매가 | 한 번에 가장 비싸게 판매한 아이템 가격 |
| 최종 상환 단계 | 마지막으로 도달한 상환 단계 |

## 17. UI 흐름

게임 흐름에 따라 UI 표시 정보가 달라진다.

| 상황 | 주요 표시 정보 |
| --- | --- |
| 기본 대기 | 보유 메소, 현재 날짜/시각, 다음 상환까지 남은 큐브 개수 |
| 아이템 선택 | 구매 가능한 아이템, 구매 비용, 장비 종류 |
| 큐브 사용 전 | 현재 아이템, 큐브 사용 버튼, 예상 판매가 없음 |
| 스크래치 중 | 잠재능력 3줄, 현재 예상 판매가, 스크래치 진행 상태 |
| 결과 완료 | 최종 판매가, 판매하기, 한 번 더 사용하기 |
| 파괴 | Destroyed 표시, 확인 버튼 |
| 빚 상환 | 상환 금액, 보유 메소, 상환 결과 |
| 게임 오버 | 최종 결과, 재시작 버튼 |
