## 1\. 개요

**큐브 스크래치**는 플레이어가 큐브를 사용한 뒤, 은박으로 가려진 잠재능력 3줄을 직접 긁어 옵션을 확인하는 시스템이다.

기존 큐브 결과 확인이 단순히 결과창을 보는 방식이라면, 본 시스템은 플레이어가 직접 잠재능력 줄을 하나씩 공개하도록 만들어 **복권을 긁는 듯한 기대감과 긴장감**을 제공한다.

플레이어는 잠재능력 3줄을 모두 공개한 뒤, 결과에 따라 현재 아이템을 판매하거나 한 번 더 큐브를 사용할지 판단한다.

\---

## 2\. 목표

큐브 스크래치의 목표는 다음과 같다.

|목표|설명|
|-|-|
|직접 조작감 제공|큐브 결과를 즉시 보여주는 것이 아니라, 플레이어가 직접 긁어 확인하게 한다.|
|기대감 형성|잠재능력 3줄이 순차적으로 공개되며 좋은 옵션을 기대하게 만든다.|
|긴장감 형성|파괴 항목이 등장할 수 있어 마지막 줄까지 긴장감을 유지한다.|
|결과 피드백 강화|유효 옵션, 이탈 옵션, 대박 조합, 파괴 여부에 따라 다른 연출을 제공한다.|
|선택 유도|결과 확인 후 판매할지, 한 번 더 큐브를 사용할지 고민하게 만든다.|

\---

## 3\. 진입 조건

큐브 스크래치 화면은 플레이어가 큐브 사용을 실행했을 때 진입한다.

|조건|내용|
|-|-|
|대상 기본 아이템 존재|큐브를 사용할 기본 아이템이 존재해야 한다.|
|큐브 사용 가능 상태|큐브를 보유하여 외부 로직에서 큐브 사용 가능 판정을 받은 상태여야 한다.|
|큐브 사용 실행|플레이어가 큐브 사용 버튼을 누르면 스크래치 팝업이 열린다.|
|결과 생성|큐브 사용 시점에 잠재능력 3줄 결과가 내부적으로 생성된다.|

> 이 문서에서는 큐브 사용 비용, 보유 메소, 빚 상환 여부는 다루지 않는다.
해당 내용은 재화 및 빚 상환 관련 문서에서 별도로 정리한다.
>

## 4\. 화면 구성

큐브 스크래치 화면은 큐브 사용 결과를 확인하는 전용 팝업 형태로 구성한다.

|영역|내용|
|-|-|
|상단 영역|큐브 이름, 큐브 아이콘|
|중앙 영역|현재 아이템 아이콘|
|결과 영역|은박으로 가려진 잠재능력 3줄|
|판매가 영역|결과 영역 오른쪽에 현재 예상 판매가 표시|
|하단 버튼 영역|판매하기 / 한 번 더 사용하기 / 확인 버튼|
|우측 상단|현재 메소 표시|
|좌측 상단|현재 날짜 / 시각 표시. 그 다음 줄에 상환까지 남은 큐브 개수 표시|

### 필수 표시 정보

|정보|설명|
|-|-|
|현재 아이템|큐브를 사용 중인 대상 아이템|
|잠재능력 3줄|스크래치로 공개할 결과 영역|
|현재 예상 판매가|3줄 공개 완료 후 표시되는 예상 판매가|
|판매하기 버튼|현재 아이템을 판매하는 버튼|
|한 번 더 사용하기 버튼|같은 아이템에 큐브를 다시 사용하는 버튼|
|확인 버튼|파괴 결과 확인 또는 팝업 종료용 버튼|

\---

## 5\. 잠재능력 결과 데이터

큐브 사용 시 내부적으로 잠재능력 3줄 결과가 생성된다.

스크래치 화면은 생성된 결과를 즉시 공개하지 않고, 은박으로 가린 상태로 표시한다.

|데이터|설명|
|-|-|
|등급|레어 / 에픽 / 유니크 / 레전더리|
|옵션 1|첫 번째 잠재능력 결과|
|옵션 2|두 번째 잠재능력 결과|
|옵션 3|세 번째 잠재능력 결과|
|옵션 타입|주요 옵션 / 잡옵 / 파괴|
|이탈 여부|일반 수치인지 이탈 수치인지 구분|
|파괴 여부|3줄 중 파괴 항목이 포함되었는지 여부|

\---

## 6\. 스크래치 방식

잠재능력 3줄은 처음부터 공개되지 않고, 은박으로 덮인 상태로 표시된다.

플레이어는 각 줄을 직접 긁어 잠재능력을 확인한다.

|항목|내용|
|-|-|
|기본 방식|순차 공개 방식|
|조작 방식|클릭 또는 짧은 드래그|
|공개 연출|은박이 벗겨지며 옵션 텍스트 등장|
|완료 조건|3줄이 모두 공개되면 결과 판정 진행|

\---

## 7\. 공개 흐름

큐브 스크래치의 기본 흐름은 다음과 같다.

```
1. 플레이어가 큐브 사용 버튼을 누른다.
2. 큐브 스크래치 팝업이 열린다.
3. 잠재능력 3줄이 은박으로 가려진 상태로 표시된다.
4. 플레이어가 첫 번째 줄을 긁는다.
5. 첫 번째 옵션이 공개되고, 결과에 맞는 피드백이 출력된다.
6. 플레이어가 두 번째 줄을 긁는다.
7. 두 번째 옵션이 공개되고, 결과에 맞는 피드백이 출력된다.
8. 플레이어가 세 번째 줄을 긁는다.
9. 세 번째 옵션이 공개되고, 결과에 맞는 피드백이 출력된다.
10. 3줄 결과를 종합해 최종 판정을 진행한다.
11. 파괴 여부와 예상 판매가를 표시한다.
12. 결과에 따라 판매하기 / 한 번 더 사용하기 / 확인 버튼을 활성화한다.
```

\---

## 8\. 공개 피드백

각 옵션이 공개될 때 결과에 따라 다른 피드백을 제공한다.

|공개 결과|피드백|
|-|-|
|잡옵|기본 공개 효과, 꽝 효과음|
|유효 옵션|작은 반짝임, 긍정적인 효과음|
|이탈 옵션|강한 반짝임, 특수 효과음|
|유효 옵션 2줄 연속|기대감을 높이는 효과음|
|유효 옵션 3줄|대박 문구, 강조 연출|
|동일 주스탯 3줄|특수 대박 연출|
|공격 계열 3줄|특수 대박 연출|
|파괴|화면 어두워짐, 아이템 파괴, 경고음|
|레전더리 등급|강한 빛, 화면 강조|

### 피드백 방향

* 좋은 옵션이 나왔을 때는 즉시 긍정적인 반응을 준다.
* 이탈 옵션은 일반 유효 옵션보다 더 강한 연출을 사용한다.
* 파괴 항목은 다른 옵션과 확실히 구분되도록 어둡고 불안한 연출을 사용한다.
* 세 번째 줄에서 대박 조합이 완성되면 가장 강한 피드백을 제공한다.

\---

## 9\. 파괴 처리

파괴는 잠재능력 3줄 중 하나라도 파괴 항목이 등장했을 때 발생한다.

파괴가 발생하면 해당 아이템은 판매할 수 없으며, 스크래치 화면은 파괴 상태로 전환된다.

|항목|내용|
|-|-|
|발생 조건|잠재능력 3줄 중 파괴 항목이 1개 이상 포함|
|표시 방식|해당 줄에 파괴 텍스트 또는 전용 아이콘 표시|
|연출|화면 어두워짐, 아이템 균열, 경고음, 파편 효과|
|결과 상태|아이템 파괴 상태로 전환|
|판매하기|비활성화|
|한 번 더 사용하기|비활성화|
|확인|활성화|
|이후 흐름|확인 시 팝업 종료 후 다음 아이템 선택 흐름으로 이동|

### 처리 기준

파괴 항목은 다른 옵션보다 우선한다.

따라서 3줄 중 좋은 옵션이나 이탈 옵션이 포함되어 있더라도, 파괴 항목이 하나라도 존재하면 최종 결과는 파괴로 처리한다.

예시:

STR: 12%

보스 공격력 30%

파괴

위 결과는 좋은 이탈 옵션이 포함되어 있지만, 파괴 항목이 있으므로 아이템은 파괴된다.

## 10\. 결과 완료 상태

잠재능력 3줄이 모두 공개되면 결과 완료 상태로 전환된다.

결과 완료 상태에서는 파괴 여부에 따라 표시 정보와 버튼 상태가 달라진다.

|결과|처리|
|-|-|
|파괴 없음|예상 판매가 표시, 판매하기 / 한 번 더 사용하기 활성화|
|파괴 있음|판매가 미표시 또는 0 표시, 판매하기 / 한 번 더 사용하기 비활성화|
|대박 결과|대박 문구와 강조 연출 출력|
|이탈 포함|판매가 상승 강조 연출 출력|
|일반 결과|기본 결과 표시|

### 예상 판매가 표시

1줄이 공개될 때마다 오른쪽의 예상 판매가가 즉시 상승한다.

3줄 공개가 완료되면 판매가 계산 공식을 통해 현재 예상 판매가를 받아와 표시한다.

\---

## 11\. 버튼 상태

스크래치 진행 상태에 따라 버튼 활성화 여부가 달라진다.

|상태|판매하기|한 번 더 사용하기|확인|
|-|-|-|-|
|스크래치 전|비활성화|비활성화|비활성화|
|1줄 공개|비활성화|비활성화|비활성화|
|2줄 공개|비활성화|비활성화|비활성화|
|3줄 공개 완료 / 정상|활성화|활성화|선택적으로 활성화|
|3줄 공개 완료 / 파괴|비활성화|비활성화|활성화|

### 버튼 처리 기준

* 스크래치가 완료되기 전까지는 판매하거나 다시 큐브를 사용할 수 없다.
* 3줄이 모두 공개된 뒤에만 판매하기와 한 번 더 사용하기를 선택할 수 있다.
* 파괴 결과가 발생하면 판매하기와 한 번 더 사용하기는 비활성화된다.
* 파괴 결과에서는 확인 버튼을 눌러 팝업을 종료한다.

\---

## 12\. 외부 연동

큐브 스크래치 화면은 다른 기능과 연결되지만, 해당 기능의 세부 규칙을 직접 관리하지 않는다.

|연동 대상|처리 방식|
|-|-|
|아이템|현재 아이템 정보를 받아와 화면에 표시한다.|
|잠재능력 생성|생성된 3줄 결과 데이터를 받아 표시한다.|
|판매가 계산|3줄 공개 완료 후 예상 판매가 값을 받아 표시한다.|
|판매|판매하기 버튼 클릭 시 판매 요청 이벤트를 전달한다.|
|재화|큐브 비용 차감, 판매 수익 지급은 외부 재화 로직에서 처리한다.|
