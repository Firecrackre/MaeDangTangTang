## 1. 개요

**판매가 계산**은 큐브 사용으로 생성된 잠재능력 결과를 바탕으로,

현재 아이템의 예상 판매가를 산정하는 규칙이다.

아이템의 판매가는 기본 판매가에 잠재능력 옵션 보너스,

조합 보너스, 등급 배수, 이탈 배수를 반영해 결정된다.

단, 모든 주요 옵션이 항상 높은 가치를 가지는 것은 아니다.

장비 종류에 따라 어울리는 스탯과 공격 계열이 다르며, 현재 아이템과 맞는 옵션만 **유효 옵션**으로 계산한다.

플레이어는 큐브 스크래치 화면에서 잠재능력 3줄을 한 줄씩 공개하며,

공개된 결과에 따라 오른쪽의 현재 예상 판매가가 상승하는 것을 확인한다.

---

## 2. 목표

판매가 계산의 목표는 다음과 같다.

| 목표 | 설명 |
| --- | --- |
| 결과 가치 반영 | 잠재능력 결과에 따라 아이템 판매가가 달라지도록 한다. |
| 장비별 유효 옵션 반영 | 장비 종류와 맞는 스탯/공격 계열만 높은 가치로 계산한다. |
| 직관적인 보상 제공 | 좋은 옵션이 많을수록 판매가가 확실히 상승하게 한다. |
| 대박 결과 강조 | 유효 주스탯 3줄, 유효 공격 계열 3줄, 이탈 옵션 등 희귀 조합의 가치를 크게 반영한다. |
| 선택 판단 지원 | 플레이어가 현재 판매가를 보고 판매 또는 재큐브를 고민하게 한다. |
| 스크래치 연동 | 한 줄씩 공개될 때마다 예상 판매가가 상승하는 피드백을 제공한다. |
| 메소 획득 쾌감 제공 | 판매가가 수만~수백만 단위로 상승하며 메이플스토리 특유의 메소 보상감을 느끼게 한다. |

---

## 3. 계산 흐름

판매가 계산은 잠재능력 판정이 완료된 결과 데이터를 기준으로 진행한다.

```
1. 현재 아이템의 기본 판매가를 불러온다.
2. 현재 아이템의 장비 종류를 확인한다.
3. 장비 종류에 맞는 유효 스탯과 유효 공격 계열을 확인한다.
4. 공개된 잠재능력 옵션을 확인한다.
5. 각 옵션이 현재 아이템에 유효한 옵션인지 판정한다.
6. 각 옵션의 옵션 보너스를 계산한다.
7. 유효 옵션 조합에 따른 조합 보너스를 계산한다.
8. 잠재능력 등급 배수를 적용한다.
9. 유효 이탈 옵션 개수에 따른 이탈 배수를 적용한다.
10. 파괴 여부를 확인한다.
11. 파괴가 없으면 최종 판매가를 산정한다.
12. 파괴가 있으면 판매가를 Destroyed로 표기한다.
13. 계산된 판매가를 큐브 스크래치 화면에 전달한다.
```

---

## 4. 기본 계산식

판매가는 다음 구조를 따른다.

```
최종 판매가 =
(아이템 기본 판매가 + 옵션 보너스 총합 + 조합 보너스) × 등급 배수 × 이탈 배수
```

단, 파괴 항목이 포함된 경우 판매가 계산을 진행하지 않고 파괴 처리한다.

```
파괴 항목 포함 시 =
판매가 계산 중단, Destroyed 표기
```

## 5. 판매가 구성 요소

판매가는 다음 요소로 구성된다.

| 요소 | 설명 |
| --- | --- |
| 아이템 기본 판매가 | 아이템이 기본적으로 가진 판매 가치 |
| 장비별 유효 옵션 | 현재 장비와 맞는 스탯/공격 계열 여부 |
| 옵션 보너스 | 각 잠재능력 옵션에 따라 추가되는 값 |
| 조합 보너스 | 특정 유효 옵션 조합이 완성되었을 때 추가되는 값 |
| 등급 배수 | 레어 / 에픽 / 유니크 / 레전더리 등급에 따른 배수 |
| 이탈 배수 | 유효 이탈 옵션 개수에 따라 적용되는 추가 배수 |
| 파괴 여부 | 파괴 항목이 있으면 판매가 계산 무효 |

## 6. 메소 단위 기준

| 결과 구간 | 판매가 규모 |
| --- | --- |
| 낮은 결과 | 25,000 ~ 100,000 메소 |
| 일반 결과 | 100,000 ~ 250,000 메소 |
| 성공 결과 | 250,000 ~ 500,000 메소 |
| 대박 결과 | 500,000 ~ 2,500,000 메소 |
| 초대박 결과 | 2,500,000 메소 이상 |

> 위 구간은 플레이어가 판매가 상승을 직관적으로 체감하도록 하기 위한 초기 기준안이다.
> 
> 
> 실제 수치는 아이템 가격, 큐브 비용, 빚 상환 금액과 함께 조정한다.
>

## 7. 아이템 기본 판매가

아이템 기본 판매가는 아이템 데이터에서 받아온다.

판매가 계산 문서에서는 아이템의 종류나 구매 가격을 직접 관리하지 않고,

현재 아이템이 가진 **기본 판매가 값**만 참조한다.

| 항목 | 설명 |
| --- | --- |
| 참조 대상 | 현재 큐브를 사용 중인 아이템 |
| 사용 값 | 아이템 기본 판매가 |
| 관리 위치 | 아이템 관련 문서 |
| 판매가 계산 내 역할 | 최종 판매가의 기준값 |

### 기본 판매가 예시

| 아이템 유형 | 기본 판매가 예시 |
| --- | --- |
| 저가 아이템 | 25,000 메소 |
| 중가 아이템 | 50,000 메소 |
| 고가 아이템 | 100,000 메소 |

> 위 수치는 초기 기준안이며, 아이템 문서와 밸런싱 결과에 따라 조정한다.
>

## 8. 장비별 유효 옵션

주요 옵션이라도 현재 아이템의 장비 종류와 맞지 않으면 높은 가치로 계산하지 않는다.

현재 아이템과 맞는 스탯 또는 공격 계열이 붙었을 때만 **유효 옵션**으로 처리한다.

### 장비별 유효 스탯

| 장비 종류 | 유효 스탯 |
| --- | --- |
| 검 | STR |
| 활 | DEX |
| 스태프 | INT |
| 완드 | INT |
| 단도 | LUK |

### 장비별 유효 공격 계열

| 장비 종류 | 유효 공격 계열 |
| --- | --- |
| 검 | 공격력, 보스 공격력 |
| 활 | 공격력, 보스 공격력 |
| 스태프 | 마력, 보스 공격력 |
| 완드 | 마력, 보스 공격력 |
| 단도 | 공격력, 보스 공격력 |

### 유효 옵션 판정 기준

| 옵션 상태 | 처리 |
| --- | --- |
| 장비와 스탯이 맞음 | 유효 주스탯 |
| 장비와 공격 계열이 맞음 | 유효 공격 계열 |
| 보스 공격력 | 모든 장비에서 유효 |
| 장비와 스탯이 맞지 않음 | 비유효 주요 옵션 |
| 장비와 공격 계열이 맞지 않음 | 비유효 주요 옵션 |
| 잡옵 | 잡옵 |
| 파괴 | 파괴 |

## 9. 옵션 보너스

옵션 보너스는 잠재능력 각 줄에 붙은 옵션에 따라 더해지는 값이다.

같은 주요 옵션이라도 현재 장비에 맞는 경우와 맞지 않는 경우의 가치가 다르다.

| 옵션 타입 | 보너스 기준 |
| --- | --- |
| 유효 주스탯 | 높은 판매가 보너스 |
| 유효 공격 계열 | 높은 판매가 보너스 |
| 비유효 주요 옵션 | 낮은 판매가 보너스 |
| 잡옵 | 매우 낮은 판매가 보너스 |
| 파괴 | 판매가 계산 무효 |

### 유효 주요 옵션 보너스 기준안

| 옵션 분류 | 옵션 | 보너스 |
| --- | --- | --- |
| 유효 주스탯 | STR / DEX / INT / LUK | +15,000 |
| 유효 공격 계열 | 공격력 / 마력 | +20,000 |
| 유효 공격 계열 | 보스 공격력 | +25,000 |

### 비유효 주요 옵션 보너스 기준안

| 옵션 분류 | 처리 | 보너스 |
| --- | --- | --- |
| 비유효 주스탯 | 장비와 맞지 않는 STR / DEX / INT / LUK | +5,000 |
| 비유효 공격 계열 | 장비와 맞지 않는 공격력 / 마력 | +5,000 |

### 잡옵 보너스 기준안

| 옵션 | 보너스 |
| --- | --- |
| 수면 효율 | +100 |
| 이동속도 | +200 |
| 점프력 | +300 |
| 언어 능력 | +400 |
| 친화력 | +500 |

> 옵션 보너스는 초기 밸런싱 기준안이다.
> 
> 
> 플레이 테스트 후 유효 옵션과 비유효 옵션, 잡옵의 가치 차이가 충분히 느껴지도록 조정한다.
>

## 10. 조합 보너스

조합 보너스는 잠재능력 3줄의 조합이 특정 조건을 만족했을 때 추가로 부여되는 값이다.

단, 판매가 계산에서 조합 보너스는 **현재 장비 기준으로 유효한 옵션**만 반영한다.

| 조합 | 조건 | 보너스 |
| --- | --- | --- |
| 유효 주요 옵션 3줄 | 현재 장비에 유효한 주요 옵션이 3개 등장 | +50,000 |
| 유효 동일 주스탯 3줄 | 현재 장비의 유효 스탯이 3개 등장 | +75,000 |
| 유효 공격 계열 3줄 | 현재 장비에 유효한 공격 계열 옵션이 3개 등장 | +90,000 |
| 유효 주요 옵션 2줄 + 이탈 포함 | 유효 주요 옵션 2개 이상, 유효 이탈 1개 이상 | +35,000 |
| 신급 대박 | 유효 공격 계열 3줄 + 유효 이탈 3개 | +150,000 |
| 전설급 대박 | 유효 동일 주스탯 3줄 + 유효 이탈 3개 | +150,000 |

### 조합 보너스 처리 기준

- 조합 보너스는 가장 높은 조건 하나만 적용한다.
- 하위 조건과 상위 조건이 동시에 만족될 경우, 상위 조건의 보너스만 적용한다.
- 비유효 주요 옵션은 조합 보너스 조건에 포함하지 않는다.
- 잡옵은 조합 보너스 조건에 포함하지 않는다.
- 파괴 항목이 포함된 경우 조합 보너스는 적용하지 않는다.
- 잠재능력 판정에서 대박 조합으로 분류되었더라도, 판매가 계산에서는 현재 장비와 맞는 유효 옵션만 조합 보너스와 이탈 배수에 반영한다.

예시:

```
아이템: 검

STR +12%
STR +9%
STR +9%
```

위 결과는 검의 유효 스탯인 STR이 3줄 등장했으므로, **유효 동일 주스탯 3줄** 보너스를 적용한다.

```
아이템: 활

STR +12%
STR +9%
STR +9%
```

위 결과는 동일 주스탯 3줄이지만, 활의 유효 스탯은 DEX이므로 유효 동일 주스탯 3줄 보너스를 적용하지 않는다.

각 STR 옵션은 비유효 주요 옵션으로 처리한다.

## 11. 등급 배수

등급 배수는 잠재능력 등급에 따라 최종 판매가에 곱해지는 값이다.

| 등급 | 판매가 배수 |
| --- | --- |
| 레어 | x1.0 |
| 에픽 | x2.0 |
| 유니크 | x3.5 |
| 레전더리 | x5.0 |

등급이 높을수록 같은 옵션 조합이라도 더 높은 판매가를 가진다.

단, 등급이 높더라도 현재 장비와 맞지 않는 옵션만 등장했다면 판매가 상승폭은 제한된다.

---

## 12. 이탈 배수

이탈 배수는 유효 이탈 옵션 개수에 따라 최종 판매가에 추가로 곱해지는 값이다.

| 유효 이탈 옵션 개수 | 이탈 배수 |
| --- | --- |
| 0개 | x1.0 |
| 1개 | x1.3 |
| 2개 | x1.8 |
| 3개 | x2.5 |

이탈 옵션은 낮은 확률로 등장하는 고가치 옵션이지만, 판매가 계산에서는 **현재 장비에 유효한 이탈 옵션만** 이탈 배수에 반영한다.

예시:

| 아이템 | 이탈 옵션 | 처리 |
| --- | --- | --- |
| 검 | STR +12% | 유효 이탈 |
| 검 | DEX +12% | 비유효 이탈 |
| 활 | DEX +12% | 유효 이탈 |
| 활 | STR +12% | 비유효 이탈 |
| 스태프 | INT +12% | 유효 이탈 |
| 스태프 | 공격력 +9% | 비유효 이탈 |
| 스태프 | 마력 +9% | 유효 이탈 |
| 완드 | INT +12% | 유효 이탈 |
| 완드 | 마력 +9% | 유효 이탈 |
| 단도 | LUK +12% | 유효 이탈 |
| 단도 | INT +12% | 비유효 이탈 |

---

## 13. 파괴 처리

파괴 항목이 잠재능력 3줄 중 하나라도 포함되어 있으면 판매가 계산을 진행하지 않는다.

### 처리 기준

파괴는 모든 계산보다 우선한다.

따라서 레전더리 등급, 유효 이탈 옵션,

대박 조합이 함께 등장하더라도 파괴 항목이 있다면 최종 판매가는 계산하지 않는다.

---

## 14. 예상 판매가 표시

큐브 스크래치 화면에서는 잠재능력 3줄이 공개될 때마다 현재 예상 판매가가 갱신된다.

| 공개 상태 | 예상 판매가 처리 |
| --- | --- |
| 스크래치 전 | 기본값 또는 미표시 |
| 1줄 공개 | 1줄 기준 임시 예상 판매가 표시 |
| 2줄 공개 | 2줄 기준 임시 예상 판매가 표시 |
| 3줄 공개 | 최종 예상 판매가 표시 |
| 파괴 공개 | 판매가에 Destroyed 표시 |

### 표시 방식

- 유효 옵션이 공개되면 예상 판매가가 즉시 상승한다.
- 유효 이탈 옵션이 공개되면 예상 판매가 상승폭을 강조한다.
- 비유효 주요 옵션이 공개되면 낮은 폭으로 상승한다.
- 잡옵이 공개되면 낮은 폭으로 상승하거나 거의 변화하지 않는다.
- 파괴가 공개되면 예상 판매가 표시를 중단하고 Destroyed로 표기한다.
- 3줄 공개 완료 후에는 최종 판매가를 확정 표시한다.

### 임시 예상 판매가 기준

1줄 또는 2줄만 공개된 상태에서는 공개된 옵션만 기준으로 임시 예상 판매가를 표시한다.

단, 3줄 조합 보너스는 3줄이 모두 공개된 뒤에만 적용한다.

| 상태 | 적용 범위 |
| --- | --- |
| 1줄 공개 | 아이템 기본 판매가 + 1줄 옵션 보너스 |
| 2줄 공개 | 아이템 기본 판매가 + 2줄 옵션 보너스 |
| 3줄 공개 | 아이템 기본 판매가 + 3줄 옵션 보너스 + 조합 보너스 + 등급 배수 + 이탈 배수 |

## 15. 계산 예시

### 예시 1. 일반 성공 결과

```
아이템: 검
아이템 기본 판매가: 50,000 메소
등급: 에픽
옵션 1: STR +9%
옵션 2: 이동속도 +10
옵션 3: 공격력 +6%
유효 이탈 옵션: 0개
파괴 여부: 없음
```

계산:

```
기본 판매가: 50,000
옵션 보너스: STR +15,000, 이동속도 +200, 공격력 +20,000
조합 보너스: 없음
등급 배수: x2.0
이탈 배수: x1.0

최종 판매가 =
(50,000 + 15,000 + 200 + 20,000) × 2.0 × 1.0
= 170,400 메소
```

반올림 적용 시:

```
표시 판매가 = 170,000 메소
```

## 16. 반올림 처리

최종 판매가는 표시 편의성을 위해 일정 단위로 반올림할 수 있다.

| 항목 | 처리 |
| --- | --- |
| 계산 결과 | 내부적으로 정수 처리 |
| 표시 단위 | 1,000 메소 단위 |
| 추천 방식 | 1,000 메소 단위 반올림 |

예시:

```
773,500 메소 → 774,000 메소
227,500 메소 → 228,000 메소
3,937,500 메소 → 3,938,000 메소
```

> 반올림 방식은 UI 가독성과 밸런스에 따라 조정한다.

## 17. 외부 연동

판매가 계산은 다른 기능과 연결되지만, 해당 기능의 세부 처리는 직접 관리하지 않는다.

| 연동 대상 | 처리 방식 |
| --- | --- |
| 아이템 | 현재 아이템의 장비 종류와 기본 판매가를 받아온다. |
| 잠재능력 판정 | 등급, 옵션 타입, 이탈 여부, 조합 판정, 파괴 여부를 받아온다. |
| 큐브 스크래치 | 현재 예상 판매가와 최종 판매가를 전달한다. |
| 판매 | 판매하기 버튼 클릭 시 확정 판매가를 전달한다. |
| 재화 | 판매 성공 시 지급할 메소 값을 전달한다. |
| 파괴 처리 | 파괴 여부가 true일 경우 판매가 계산을 중단한다. |
